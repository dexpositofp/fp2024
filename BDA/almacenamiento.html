Ficheros

Los ficheros son la forma más básica de almacenamiento de datos. Existen varios tipos, cada uno con sus características y aplicaciones:

  • Ficheros planos: Son simples archivos de texto donde los datos se almacenan en líneas consecutivas. Estos son fáciles de usar, pero pueden volverse ineficientes cuando los datos crecen. Además, suelen ser propensos a problemas de inconsistencia y redundancia debido a la falta de control de integridad y la dificultad para mantener relaciones complejas entre datos.
  • Ficheros indexados: Utilizan índices para acelerar la búsqueda de datos. Un índice es una estructura adicional que permite acceder rápidamente a los registros sin necesidad de recorrer todo el fichero.
  • Ficheros de acceso directo: Permiten acceder a los datos directamente mediante una clave o identificador, sin necesidad de leer secuencialmente desde el inicio, lo que mejora la eficiencia en la recuperación de información.

Problemas de los Ficheros

Trabajar con ficheros, especialmente ficheros planos, puede presentar varios problemas:

  • Inconsistencia de Datos: Los datos pueden duplicarse y no ser actualizados en todas las ubicaciones, lo que lleva a inconsistencias.
  • Redundancia de Datos: Los mismos datos se almacenan en múltiples lugares, ocupando más espacio del necesario y dificultando su gestión.
  • Dificultad de Acceso: Las búsquedas pueden ser lentas y complicadas, especialmente en ficheros grandes.
  • Falta de Seguridad: Es difícil implementar controles de acceso y asegurar los datos en sistemas de ficheros simples.

Ejemplo de un fichero plano (CSV):

nombre,edad,ciudad
Juan,28,Madrid
Ana,32,Barcelona
Luis,24,Valencia

Ejemplo de un fichero indexado:

Archivo de datos:
[1] Juan,28,Madrid
[2] Ana,32,Barcelona
[3] Luis,24,Valencia

Archivo de índice:
ID   Posición
1    0x0000
2    0x0014
3    0x0028

En este ejemplo, el archivo de índice permite acceder directamente a la posición del registro en el archivo de datos, acelerando las búsquedas.

Ejemplo de un fichero de acceso directo:

Registro 1 -> clave: "1234", datos: "Juan,28,Madrid"
Registro 2 -> clave: "5678", datos: "Ana,32,Barcelona"
Registro 3 -> clave: "9101", datos: "Luis,24,Valencia"

Acceso directo utilizando la clave "5678" para obtener el registro correspondiente.

Bases de Datos

Las bases de datos son estructuras organizadas de datos que permiten su almacenamiento, recuperación y manipulación eficiente. A diferencia de los ficheros, las bases de datos permiten mantener la integridad de los datos y soportan operaciones más complejas.

Existen varios tipos de bases de datos según el modelo de datos:

  • Relacionales: Utilizan tablas para representar los datos y sus relaciones. Son las más comunes y utilizan SQL para la gestión de datos.
  • NoSQL: Incluyen modelos como documentos, clave-valor, columnas y grafos. Son más flexibles y escalables, especialmente para grandes volúmenes de datos no estructurados.
  • Jerárquicas: Organizan los datos en una estructura de árbol, donde cada nodo tiene un solo padre y múltiples hijos. Esto puede ser útil para datos con relaciones jerárquicas naturales.
  • En red: Similar al modelo jerárquico, pero permite relaciones más complejas entre los datos, con nodos que pueden tener múltiples padres.
  • Orientadas a objetos: Combinan conceptos de bases de datos con la programación orientada a objetos, permitiendo almacenar objetos completos en lugar de solo datos primitivos.

Ejemplo de una consulta SQL básica en una base de datos relacional:

SELECT nombre, edad 
FROM usuarios 
WHERE ciudad = 'Madrid';

Sistemas Gestores de Bases de Datos (SGBD)

Los Sistemas Gestores de Bases de Datos (SGBD) son herramientas que permiten gestionar bases de datos de manera eficiente. Sus principales funciones incluyen:

  • Definición de datos: Creación y modificación de estructuras de datos.
  • Manipulación de datos: Inserción, actualización, eliminación y consulta de datos.
  • Control de acceso: Autenticación y autorización de usuarios.
  • Mantenimiento: Copias de seguridad, recuperación de datos, etc.

Los componentes clave de un SGBD incluyen:

  • Motor de base de datos: Maneja el almacenamiento, recuperación y actualización de datos.
  • Lenguaje de consulta: Interfaz para interactuar con los datos (SQL es el más común).
  • Sistema de gestión de transacciones: Asegura que las operaciones de la base de datos se completen correctamente.

Niveles de ANSI/SPARC y la Independencia de Datos

El modelo ANSI/SPARC define tres niveles de abstracción en una base de datos:

  • Nivel Interno: Define cómo se almacenan los datos en el sistema.
  • Nivel Conceptual: Describe la estructura lógica global de la base de datos, sin preocuparse por cómo se almacenan los datos.
  • Nivel Externo: Define las vistas individuales de los datos para diferentes usuarios.

La independencia de datos se refiere a la capacidad de modificar la estructura de la base de datos sin afectar a las aplicaciones que la utilizan. Esto se logra a través de la separación de estos niveles.

Perfiles de Usuarios

En el contexto de un SGBD, existen varios perfiles de usuarios con diferentes niveles de acceso y responsabilidades:

  • Administradores de Base de Datos (DBA): Encargados de la configuración, mantenimiento y seguridad del SGBD.
  • Desarrolladores: Diseñan y desarrollan aplicaciones que interactúan con la base de datos.
  • Usuarios Finales: Interactúan con la base de datos a través de las aplicaciones, realizando consultas y actualizaciones según sus permisos.

SGBD Comerciales y Libres

Los SGBD pueden ser clasificados como comerciales o libres:

  • Comerciales: Son desarrollados por empresas y requieren licencias pagadas. Ejemplos: Oracle, Microsoft SQL Server. Estos suelen ofrecer soporte técnico y características avanzadas.
  • Libres: Son de código abierto y generalmente gratuitos. Ejemplos: MySQL, PostgreSQL. Son altamente personalizables y tienen comunidades activas que ofrecen soporte.

Bases de Datos Centralizadas y Distribuidas

Las bases de datos pueden ser centralizadas o distribuidas:

  • Centralizadas: Todos los datos se almacenan en un solo lugar, lo que facilita la gestión y el control, pero puede presentar problemas de escalabilidad y disponibilidad.
  • Distribuidas: Los datos se reparten en múltiples ubicaciones geográficas. Esto mejora la disponibilidad y el rendimiento, pero puede ser más complejo de gestionar.

La fragmentación es una técnica usada en bases de datos distribuidas para dividir una base de datos en fragmentos más pequeños que pueden almacenarse en diferentes ubicaciones. Hay varios tipos de fragmentación:

  • Fragmentación Horizontal: Divide los datos en subgrupos de registros. Por ejemplo, una base de datos de clientes podría fragmentarse en función de la región geográfica.
  • Fragmentación Vertical: Divide los datos en subgrupos de atributos. Por ejemplo, una base de datos de empleados podría dividirse en una fragmentación que contenga información personal y otra con información de salarios.
  • Fragmentación Mixta: Combina fragmentación horizontal y vertical para una mayor flexibilidad y eficiencia.

Bases de Datos Relacionales

Las bases de datos relacionales están fundamentadas en la teoría de conjuntos y el álgebra relacional, lo que les otorga una base matemática sólida para la manipulación y consulta de datos. Se introdujeron por primera vez en la década de los 70, con el trabajo de Edgar F. Codd, quien estableció los principios que siguen rigiendo este tipo de sistemas hasta la fecha.

Base Matemática: Álgebra Relacional

El álgebra relacional es un lenguaje formal que proporciona un conjunto de operaciones para manipular las relaciones (tablas) en una base de datos relacional. Las operaciones básicas incluyen:

  • Selección (σ): Permite obtener un subconjunto de filas de una tabla que cumplen una condición específica.
  • Proyección (π): Selecciona columnas específicas de una tabla.
  • Unión (∪): Combina las filas de dos tablas con la misma estructura.
  • Intersección (∩): Retorna las filas que son comunes en dos tablas.
  • Diferencia (-): Retorna las filas que están en la primera tabla pero no en la segunda.
  • Producto cartesiano (×): Combina cada fila de la primera tabla con cada fila de la segunda tabla.
  • Join (⨝): Combina filas de dos tablas basadas en una condición lógica.

Principios Básicos

Las bases de datos relacionales se construyen a partir de tablas que representan entidades. Cada tabla está formada por filas (también llamadas registros o tuplas) y columnas (llamadas atributos o campos).

Los principios fundamentales de una base de datos relacional son:

  • Integridad de Entidad: Cada tabla debe tener una clave primaria que identifique de manera única cada fila. Esto asegura que no existan registros duplicados.
  • Integridad Referencial: Si una tabla tiene una clave ajena que hace referencia a la clave primaria de otra tabla, el valor de la clave ajena debe existir en la tabla referenciada. Esto mantiene la coherencia entre las tablas.
  • Normalización: Es un proceso que organiza los datos para reducir la redundancia y mejorar la integridad. Las bases de datos bien diseñadas suelen seguir diversas formas normales.

Conceptos Clave

A continuación, se describen algunos conceptos esenciales en las bases de datos relacionales:

  • Dominio: Es el conjunto de valores permitidos que un atributo puede tomar. Por ejemplo, el dominio de un atributo "Edad" podría ser los números enteros positivos.
  • Tupla: Es una fila en una tabla, que representa una instancia de una entidad. Cada tupla está compuesta por un conjunto de valores.
  • Clave Primaria: Es un atributo o un conjunto de atributos que identifica de manera única a cada tupla en una tabla. Por ejemplo, el número de DNI en una tabla de personas.
  • Clave Foránea: Es un atributo en una tabla que se refiere a la clave primaria de otra tabla, estableciendo una relación entre las dos tablas.

Ejemplo de Tablas Relacionales

Supongamos que tenemos dos tablas, Personas y Direcciones, relacionadas por una clave foránea:

        Personas (id_persona, nombre, apellido, id_direccion)
        -------------------------------------------
        1, Juan, Pérez, 101
        2, María, Gómez, 102
        3, Pedro, López, 101

        Direcciones (id_direccion, calle, ciudad)
        -------------------------------------------
        101, Calle Mayor, Madrid
        102, Avenida Central, Barcelona

En este ejemplo, id_persona es la clave primaria de la tabla Personas, mientras que id_direccion es una clave foránea en Personas, que hace referencia a la clave primaria de la tabla Direcciones.

Sistemas Gestores de Bases de Datos Relacionales (RDBMS)

Los sistemas de gestión de bases de datos relacionales son software que permiten crear, gestionar y consultar bases de datos relacionales. Entre los más utilizados se encuentran:

  • MySQL: Uno de los RDBMS más populares y utilizados en aplicaciones web.
  • PostgreSQL: Un sistema gestor de bases de datos relacionales avanzado con soporte para SQL y características adicionales.
  • Oracle Database: Un sistema empresarial con características robustas para grandes organizaciones.
  • Microsoft SQL Server: Muy utilizado en entornos empresariales que usan software de Microsoft.

Ventajas de las Bases de Datos Relacionales

Las bases de datos relacionales resuelven muchos de los problemas del manejo tradicional de ficheros, como la inconsistencia de datos, la redundancia y la dificultad para acceder a la información. Entre sus principales ventajas destacan:

  • Atomicidad: Garantizan que las transacciones se completen de manera íntegra; si una parte falla, la transacción no se realiza.
  • Concurrencia: Permiten que múltiples usuarios accedan a los datos simultáneamente sin generar conflictos.
  • Integridad de los Datos: Las restricciones y reglas de integridad aseguran que los datos se mantengan consistentes.
  • Escalabilidad: Son capaces de manejar grandes volúmenes de datos y usuarios sin perder rendimiento.
