Presentación del Curso

Bienvenidos al curso de Bases de Datos. En este curso, exploraremos los fundamentos y las aplicaciones de las bases de datos, desde su diseño lógico y físico hasta el uso de lenguajes de consulta y manipulación de datos. Aprenderemos sobre los distintos tipos de sistemas de gestión de bases de datos (relacionales y NoSQL).

Además, conocerás las mejores prácticas para el almacenamiento, recuperación y seguridad de la información. Este curso está diseñado para proporcionarte una base sólida y las habilidades necesarias para trabajar con bases de datos en diversos entornos profesionales.

Objetivos del Curso

  • Comprender los conceptos fundamentales de las bases de datos y su importancia en el manejo de la información.
  • Diseñar bases de datos utilizando el modelo entidad-relación (E/R) y su paso al modelo relacional.
  • Dominar el uso de lenguajes de consulta (SQL) para manipular y extraer información de bases de datos.
  • Aplicar los principios de normalización y desnormalización en el diseño de bases de datos.
  • Implementar bases de datos relacionales y gestionar su rendimiento, seguridad y mantenimiento.
  • Conocer y manejar herramientas comerciales y de código abierto para la gestión de bases de datos.
  • Identificar y solucionar problemas comunes de gestión de bases de datos, incluyendo concurrencia, atomicidad y recuperación de desastres.

Materiales y Recursos

Para aprovechar al máximo este curso, recomendamos tener acceso a las siguientes herramientas y recursos:

  • Sistemas gestores de bases de datos: MySQL, Oracle, Mongodb.
  • Herramientas de diseño de diagramas.
  • Herramientas de edición de código

Estructura del Curso

El curso de Bases de Datos está dividido en varias secciones, cada una de las cuales aborda un aspecto fundamental del diseño, gestión y uso de bases de datos.

A continuación, se presenta una breve descripción de cada una de las secciones:

  • Almacenamiento: Esta unidad cubre los conceptos básicos del almacenamiento de datos, incluyendo ficheros y bases de datos. Se analizan los diferentes tipos de ficheros (planos, indexados, acceso directo) y los problemas asociados con su uso.
  • Diseño lógico: Se enfoca en el diseño lógico de bases de datos, abarcando conceptos como entidades, relaciones, cardinalidades y normalización. Aquí se introduce el diagrama entidad-relación (E/R) y su paso al modelo relacional.
  • Diseño físico: Esta sección trata sobre la implementación física de bases de datos. Se explican los métodos de almacenamiento, índices y técnicas para optimizar el rendimiento y la eficiencia.
  • DDL (Lenguaje de Definición de Datos): En esta unidad, se estudian los comandos de SQL que se utilizan para definir y modificar la estructura de las bases de datos. Se incluye la creación de tablas, índices, y restricciones.
  • DML (Lenguaje de Manipulación de Datos): Aquí se aprenderán los comandos para insertar, actualizar, eliminar y consultar datos dentro de las bases de datos. Se pondrá especial énfasis en las sentencias SELECT, INSERT, UPDATE y DELETE.
  • DCL/TCL (Lenguajes de Control de Datos y Control de Transacciones): Esta unidad cubre los lenguajes y comandos necesarios para gestionar permisos de acceso y control de transacciones dentro de una base de datos, incluyendo la atomicidad y la concurrencia.
  • Utilidades: Se abordarán las herramientas y utilidades comunes para la administración de bases de datos, tanto comerciales como de código abierto.
  • PL/SQL: Se introducirá la programación en PL/SQL, un lenguaje de programación procedural específico para bases de datos relacionales que permite crear procedimientos, funciones y triggers.
  • NoSQL: Finalmente, esta sección explorará las bases de datos NoSQL, que se utilizan para gestionar grandes volúmenes de datos no estructurados o semi-estructurados. Se analizarán los diferentes tipos de bases de datos NoSQL y su aplicación en la industria.

Evaluación

La evaluación del curso se basará en la participación en clase, ejercicios prácticos, y un proyecto final donde demostrarás tus habilidades adquiridas.

Dentro de cada unidad formativa, se realizarán difentes ejercicios prácticos de seguimiento de la unidad. Dichos ejercicios no son evaluables, pero serán corregidos en el aula y es muy recomendable realizarlos para poder superar dicha unidad.

La nota de cada unidad formativa se obtendrá de una prueba objetiva individual (examen), que será el 60% de la nota de la unidad formativa y el 40% será del proyecto propuesto para dicha unidad.

Importante: La nota mínima de los exámenes para poder hacer media es de 5. En cualquier otro caso, se deberá recuperar la unidad formativa a final de curso.

Para obtener la nota del proyecto, es imprescindible superar una prueba de validación individual. Esta prueba tiene como objetivo asegurar que cada estudiante ha comprendido y dominado los conceptos clave desarrollados durante el proyecto.

Importante: Si no se supera esta prueba de validación, la calificación del proyecto será un 0, independientemente del trabajo realizado en grupo o de la calidad del proyecto entregado.

Por lo tanto, es crucial que cada miembro del equipo domine los aspectos fundamentales y sea capaz de defender su trabajo individualmente.

Para más información, se debe consultar la programación didáctica de la asignatura.
