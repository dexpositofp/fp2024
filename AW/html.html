Introduction to HTML

HTML (short for HyperText Markup Language) is the standard language used to create and design web pages. HTML defines the structure of a web document and, along with CSS and JavaScript, forms the core of most websites we visit.

What is HTML?

HTML is a markup language that uses a series of elements or tags to define the content of a web page. Each of these elements describes different types of content, such as text, images, links, lists, forms, etc. For example, the tag: <p> defines a paragraph of text, while: <img> inserts an image into the page.

Meaning of HTML

The acronym HTML breaks down as follows:

  • H - Hyper: Refers to hyperlinks, which allow users to navigate from one page to another on the web.
  • T - Text: The main content presented on the web, which can include text, images, videos, etc.
  • M - Markup: Refers to the marking up of documents, i.e., the structure and organization of content using tags.
  • L - Language: Indicates that HTML is a language used to define how content is structured and presented on the web.

History and Evolution of HTML

HTML was developed by Tim Berners-Lee in 1991 and has evolved significantly since its early versions. Over time, new versions have introduced more tags and functionalities, adapting to the changing needs of the web. The most recent version is HTML5, which offers improved support for multimedia, graphics, and modern APIs.

Why is HTML Important?

HTML is fundamental for web development because it defines the structure and content of web pages. Without HTML, browsers would not be able to interpret and display content consistently. Additionally, HTML is the foundation upon which styles (with CSS) and interactivity (with JavaScript) are applied.

Basic HTML Example

Below is a basic example of an HTML document:


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Web Page</title>
</head>
<body>
    <h1>Hello, World!</h1>
    <p>This is my first HTML document.</p>
</body>
</html>

HTML Document Validation

Validating HTML documents is a crucial process to ensure that the code complies with specifications and standards defined by the W3C. This helps detect errors and ensures compatibility with various browsers.

There are online tools and code editors that allow for automatic validation of HTML documents, ensuring that the code is properly structured and free of errors that could affect the display and functionality of web pages.

Developers should integrate validation into their workflow to maintain the quality and consistency of HTML code throughout web application development.

Web Design Principles

Web design principles guide the creation of effective and attractive interfaces for users. Some fundamental principles include:

  • Usability: Ease of use and intuitive navigation.
  • Accessibility: Ensuring that all users, including those with disabilities, can access content equitably.
  • Readability: Clear and well-structured text for easy comprehension.
  • Consistency: Maintaining consistent visual and interaction patterns throughout the interface.
  • Aesthetics: Balance between functionality and visually appealing design.

Web developers should apply these principles to create effective and satisfying user experiences.

Usability and Accessibility

Usability refers to how easily users can interact with a web application and complete tasks. Accessibility, on the other hand, focuses on making websites accessible to all users, including those with disabilities.

To improve usability, developers should consider aspects such as intuitive navigation, page load speed, and content layout. Accessibility involves using appropriate tags, providing alternatives for multimedia content, and ensuring sufficient contrast in design.

Adhering to standards and best practices, such as W3C accessibility guidelines, helps ensure that all users can benefit from web content equitably.

HTML Syntax. Basic Tags

HTML uses a simple and consistent markup syntax to define the structure and content of a web page. The most common basic tags include:

  • <!DOCTYPE html> Defines the version of HTML used in the document.
  • <html> Defines the beginning and end of an HTML document.
  • <head> Contains the document's header information, such as metadata, links to stylesheets, and scripts.
  • <title> Defines the title of the web page that appears in the browser tab.
  • <body> Contains the main visible content of the page, including text, images, and other elements.
  • <h1> to <h6> Define headings of different levels of importance.
  • <p> Defines a paragraph of text.
  • <a> Defines a hyperlink.
  • <img> Inserts an image into the page.
  • <ul> <li> Defines unordered lists with list items.

These tags are fundamental for structuring and presenting content in a semantic manner on the web.

New HTML5 Tags

HTML5 introduced several new tags that facilitate the creation of semantic and accessible content. Some of these new tags include:

  • <header> Defines the header section of a page or section.
  • <nav> Defines navigation links.
  • <section> Represents a section of content within a document.
  • <article> Defines a self-contained piece of content, such as a blog post or news article.
  • <aside> Represents content indirectly related to the surrounding content, like a sidebar.
  • <footer> Defines the footer section of a page or section.
  • <figure> Represents a unit of content, such as an image or diagram, along with a caption.
  • <figcaption> Provides a caption for a <figure> element.
  • <main> Represents the main content of a document.

Using these new tags enhances the semantic structure of web documents, making them easier to understand for both humans and machines.

Example of Semantic Structure


<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Semantic Structure</title>
</head>
<body>
    <header>
        <h1>Main Header</h1>
        <nav>
            <ul>
                <li><a href="#home">Home</a></li>
                <li><a href="#about">About</a></li>
                <li><a href="#contact">Contact</a></li>
            </ul>
        </nav>
    </header>
    <section>
        <h2>Main Section</h2>
        <article>
            <h3>Article 1</h3>
            <p>This is an article.</p>
        </article>
        <article>
            <h3>Article 2</h3>
            <p>This is another article.</p>
        </article>
    </section>
    <aside>
        <h2>Sidebar</h2>
        <p>Additional content.</p>
    </aside>
    <footer>
        <p>Footer</p>
    </footer>
</body>
</html>

New Form Tags and Attributes

HTML5 also introduces new tags and attributes to enhance form functionality.

New Tags

  • <input type="email"> Field for entering an email address.
  • <input type="url"> Field for entering a URL.
  • <input type="tel"> Field for entering a phone number.
  • <input type="date"> Field for selecting a date.
  • <input type="range"> Slider for selecting a value within a range.

Form Example


<form>
    <label for="email">Email:</label>
    <input type="email" id="email" name="email"><br><br>

    <label for="website">Website:</label>
    <input type="url" id="website" name="website"><br><br>

    <label for="phone">Phone:</label>
    <input type="tel" id="phone" name="phone"><br><br>

    <label for="birthday">Birthday:</label>
    <input type="date" id="birthday" name="birthday"><br><br>

    <label for="volume">Volume:</label>
    <input type="range" id="volume" name="volume" min="0" max="100"><br><br>

    <button type="submit">Submit</button>
</form>

Multimedia Tags in HTML5

HTML5 introduces tags to handle multimedia content more easily and directly.

New Tags

  • <audio> Defines audio content.
  • <video> Defines video content.
  • <canvas> Used to draw graphics via scripts (like JavaScript).
  • <svg> Defines scalable vector graphics.

Example of Audio and Video


<!-- Audio -->
<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    Your browser does not support the audio element.
</audio>

<!-- Video -->
<video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    Your browser does not support the video tag.
</video>

Example of Canvas


<canvas id="myCanvas" width="200" height="100" style="border:1px solid #000000;"></canvas>
<script>

Restrictions Imposed by XHTML

XHTML (Extensible Hypertext Markup Language) is a reformulation of HTML as XML. Although XHTML follows the same semantic specifications as HTML, it imposes additional restrictions:

  • XML Well-Formed: XHTML documents must be structurally correct according to XML rules.
  • Lowercase Usage: All elements and attributes must be in lowercase.
  • Tag Closure: All tags must be properly closed.
  • Mandatory Attributes: Some elements require specific attributes to comply with XHTML syntax.

These restrictions make XHTML stricter than HTML in terms of syntax and compliance, but promote interoperability and cleaner web development.
